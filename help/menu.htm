2.4 Security & user management

User menu

Depending on the privileges of a user, he may also see the user menu – typically seen by systems administrator user. A list of users is presented as in Figure 15.

 

Figure 15 User Management view

 

The processes for adding, viewing details, editing and deleting a user are the same as for all other kinds of data. A notable exception is that passwords are hashed out and cannot be read. A systems administrator can only assign a new password but cannot recover a lost password.  The administrator can also change the role of a user from one type to another.

 

Change password menu

Figure 16 Change Password form

The change password form (Figure 16) allows a user to change their own password. The old password must be supplied first, and then the new password. If the old password is incorrect, the change will not be effected, and if the two new passwords do not match, no change will be effected. Warning/error messages are displayed to that effect.

 

2.5 Reports & Graphs

A number of reports can be derived from the application in real time based on data available at the time of report generation. The range of reports can be accessed by selecting the “Reports” menu

 

Selecting the reports menu displays a list of reports as in Figure 18. You may select any report to view by clicking on the list of reports on the left hand side.

 

Figure 18 Reports Menu

 

Some reports selected display both data and a graph. (Figure 19)

Figure 19 Reports with Graphs

 

Reports covering date range may be specified. Provision is made to make use of popup calendars (Figure 20) to ensure dates are entered correctly and easily.

Figure 20 Use of pop up calendar

 

Reports may be further filtered by selecting the range of categories of data to include or exclude. (See Figure 21) This depends on the type of report.

Figure 21 Use of Filters

A report may return many records. The user may specify and change how many records to show per page, in order to make analysis or visualization of data earlier. Further, there are navigation tools (the arrows) at the end of every display of records, to facilitate movement between the different “pages” of records returned.

Figure 22 Change number of records per page

 

 

 

 

 

 

Areas for possible future extension:

This system can be extended to have a mobile app component, so that field officers, or indeed anyone, can have real time access to the data from their mobile phones if desired. A mobile phone that has connectivity to the internet will be required. As a general guide, a phone that has a globe icon (for internet access), and that has a folder for “games” or “applications”, and usually has a color screen is a likely candidate.
