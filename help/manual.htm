2.0 Software Manual

The following documentation serves as a guide to how to use the application. This describes all features of the application, but what a user may see or interact with successfully depends on the permissions they company grants them (based on their role)

2.1 Introduction: General overview and guidelines for usage

To use this application from a PC, open your Internet browser (eg Internet Explorer, Firefox, Chrome, Safari or other preferred browser). Enter the web address of the application  in the address bar. It is currently

http://74.53.43.19/~plangh/

(note that there is no preceding www)

 

You will be presented with a screen like Figure 2. Enter your username and password. Click on the “Login” button.

Figure 2 Login screen

 

On successful login, the main view of the application is presented as in Figure 3. The main features of the application are highlighted in the figure and are described below. Note that depending on a user’s access permission and privileges, the options available, particularly the number of enabled menu items may be different, and subsequently, the ability to add, edit, delete or search may be varied.

Figure 3 Main Page

 

From this landing page, the most frequently used features of the application can be reached in one click. However, all other features can be access by using the drop down menus at the top of page.

 

 

Figure 4 List View for interacting with data

 

When a menu item or link on the landing page is selected, the user is typically presented with a list view as in Figure 4. The following illustrates a typical interaction with the system. Note however that this behavior is uniform across the application unless otherwise indicated.  It is a simple, intuitive, consistent interface provided for interacting with the data.

 

v  By clicking a menu item (see 5 in Figure 4)  at the top of the page, a table displaying the first few records of the corresponding result set is shown. The data presented gives a brief overview of the data for any one record (in this case a program area). Clicking on the “Program Area Data àProgram Area” menu item shows records for all program areas. Similarly, clicking on “Sponsored StudentàSponsored Student” will show student records. The menu items are intuitive and self explanatory. The function of each menu is nevertheless explained in Table 1. (in a later section)

v  New records can be added by clicking on the “add” link (see 7) A form will be presented for data entry.

v  To see further details for any record, click on the “view” link – the magnifying glass icon (see 4) on a particular row. The entire data on a particular entry can be viewed. Similarly, a particular row can be edited  or deleted by clicking on the appropriate icons.

v  A search facility is provided. By clicking on the “advanced search” link (see 2) specific records or groups of records matching any criteria specified by the user can be obtained.

v  Records displayed can be sorted alphabetically or reverse alphabetically by clicking on the heading of the column (see 1).  In the example in the figure, by clicking on the caption (program area/unit), the data will be sorted in alphabetical order. By clicking on the caption again, the data will be reverse sorted.

v  A user can also navigate to subsequent pages to see additional pages if there are more than a screen full of records in a result set by using navigational buttons at the bottom of the page. (see 8)

v  Data in a result set appearing on the screen can be exported to MS Excel (or any other spreadsheet) and to MS Word (or any other word processing software) by using the appropriate export link. (see 3). It can also be printed directly to a printer.

v  The user an always return to the landing page by clicking on the Plan Ghana logo at the top left of the page(see 9)

v  Some data can have related records. In the example shown in Figure 4, Program Areas have sponsored students. By clicking on the appropriate link (see 6), the user can review the associated collection of data pertaining to the selected record(program area).

 

 

View Details view

For any menu selected, a summary of transactions for that category of data is presented. To view details of a particular transaction, click on the “view” link on the first page (see 5 in Figure 4).

The resulting “view details” will look like in Figure 5. Figure 5 represents details of one row/transaction for the “Ship: Farm to Warehouse” table.

 

Figure 5 Details view

From Figure 5, it can be seen that the user can navigate back to the list, (see circled portion), or can edit the record in order to make changes to the data, or can delete this particular transaction. Note that the options available to a particular user depend on their access privileges.

 

Edit

On editing a record (either from the main list view or from the detail view) the user sees a form like in Figure 6. On completion of modifications to the data, click the “Edit” button at the bottom of the form to update the database. Without saving this way, changes will be discarded.

 

 

Figure 6 Edit view

 

The form has been created in such a way as to minimize data entry. Much data can be supplied by simply selecting appropriate values from drop down lists.  For example, in Figure 7, the user only needs to select preset or pre-supplied values from the list provided. This minimizes errors in data entry and improves integrity of data as well as speeds up data entry. Similar drop downs exist for frequently used data such as program areas, schools, regions and districts of operation.

 

Figure 7 Edit view illustrating dropdown options

 

To ensure dates are entered correctly, provision is made to make use of popup calendars. This is illustrated in Figure 8. The popup calendar will always open to show the current date if the field was blank initially, or will show the date that needs editing. The calendar has shortcuts for advancing quickly to the next month or to the next year.

Figure 8 Edit view- use of popup calendar

 

 

Deletions

Unwanted data can be deleted by clicking on the delete link (either from the main list view or from the detail view). A confirmation request is issued. The user can confirm the action or back off by clicking the “Back to List” link or by clicking on any menu item. In this application, there will rarely be the need to delete data, and it will affect data consistency. A few personnel may be granted the opportunity to delete certain types of data.

 

Add

Addition of new record is similar to editing a record except that all the fields are blank.  First choose a menu item that represents the kind of data you wish to append/add to. Next click on the “add link” (see 2 Figure 4). An appropriate form is displayed for the user to enter data as in Figure 9.

Figure 9 Add form

Required fields are marked with a red asterisk. When such required fields are not supplied, an appropriate error message is displayed such as in Figure 10.

Figure 10 Validation of input- an empty required field is flagged

 

After data entry, click on the “Add” button at the bottom of the form. On successful addition, the user is returned to the main list and a confirmation message is displayed as in Figure 11 is displayed.

 

 

Figure 11 Add confirmation

 

 

Search

A comprehensive search feature is provided to allow a user to locate records by any combination of desirable criteria. First the user must choose a menu item that represents the group of data that he wants to search. Next user clicks on the search link at the top of the displayed table. (see 3 in Figure 4). A form like Figure 12 is presented to the user. The user may enter any combination of criteria for the search and then click the search button at the end of the form.

 

Figure 12 Search view

 

It is possible to search for a range of records. For example it may be desired to find all data larger than some value or between two values. This can be accomplished by making use of the various search options for comparison. Figure 13 illustrates this.

 

Figure 13 Search options - range and combinations

 

For the specific example illustrated in Figure 13, the options are as follows:

The symbols employed have their usual meanings, namely

v  Data greater than/after user supplied parameter  use >

v  Data less than/before user supplied parameter   use <

v  Data equal to user supplied parameter   use =

v  Data greater than or equal to(on or after) user supplied parameter   use >=

v  Data less than or equal to(on or before) user supplied parameter   use <=

v  Data not equal to(any date besides ) user supplied parameter   use <>

 

Further, to obtain a range between two dates (or two values), or to enforce two conditions choose the “AND” option to ensure that the two conditions are applied. To use either of two conditions, choose the “OR” option. See the circled portions in Figure 13.

 

Note that these range choices work for all types of values –numeric, alphabetic, and date types.
